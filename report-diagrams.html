4. Leave Request Workflow
Complete flow from leave submission to final decision.
      flowchart TD
        Start(["Student submits leave request"]) -->
        Validate{"Validate input"}
        Validate -->|"Invalid"| Error["Show error message"]
        Error --> Start
        Validate -->|"Valid"| Save["Save to database
Status: pending"]
        Save --> Notify["Faculty notified via Approvals page"]
        Notify --> Review{"Faculty/Admin reviews"}

        Review -->|"Approve"| CheckBal{"Check leave balance"}

        CheckBal -->|"Sufficient"| Deduct["Deduct from balance"]
        Deduct --> Approved["Status: approved
Balance updated"]
        Approved --> CommentA["Faculty adds comment"]
        CommentA --> DoneA(["Student sees approved status"])

        CheckBal -->|"Insufficient"| RejectBal["Auto-reject"]
        RejectBal --> DoneR(["Status: rejected
Balance unchanged"])

        Review -->|"Reject"| RejectManual["Status: rejected"]
        RejectManual --> CommentR["Faculty adds reason"]
        CommentR --> DoneR

        Review -->|"Request Info"| Pending["Status: pending"]
        Pending --> Start

6. Sequence Diagram — Leave Request Interaction
Time-ordered interaction between components during a leave request.
      sequenceDiagram
        participant S as Student
        participant F as Frontend (React)
        participant B as Backend (Express)
        participant DB as Database (SQLite)

        S->>F: Fill leave form
        S->>F: Click "Analyze with AI"
        F->>B: POST /api/ai/analyze-reason
        B->>B: Analyze reason text
        B-->>F: Return suggested leave type
        F-->>S: Show suggestion

        S->>F: Submit leave request
        F->>B: POST /api/leaves
        B->>DB: Insert leave (status: pending)
        DB-->>B: Confirm insert
        B-->>F: Success response
        F-->>S: Show success message

        Note over F,DB: Faculty reviews later

        participant Fac as Faculty
        Fac->>F: View pending approvals
        F->>B: GET /api/leaves/pending
        B->>DB: Query pending leaves
        DB-->>B: Return results
        B-->>F: Display pending leaves

        Fac->>F: Click Approve
        F->>B: PUT /api/leaves/:id/approve
        B->>DB: Update status to approved
        B->>DB: Deduct from balance
        DB-->>B: Confirm
        B-->>F: Success
        F-->>Fac: Show approved
